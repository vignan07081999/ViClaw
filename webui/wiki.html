The Terminal Super Menu

Running ./viclaw from the installation directory opens the Master Interface. From here, you can manage the absolute core of the agent.

[1] Interactive CLI Chat: Drops you directly into a terminal conversation with the agent. Uses raw text streams.
[2] Start/Restart Background Daemon: Binds the AI logic to an infinite loop, exposing the WebUI on Port 8501 and allowing Telegram/Discord polling.
[4] System Diagnostics: Polls the CPU, RAM, Network, and Ollama registry for latency constraints.
[5] Agent Doctor: An autonomous script that reads the crash logs of ViClaw and attempts to self-heal the python environment or kill rogue locking processes.
[7] View Chat History & Action Logs: Dumps the SQLite memory.db directly to the console so you can read what the agent is "thinking" without a web browser.
[10] OTA Updates: Checks the connected GitHub repository (vignan07081999/ViClaw) for code changes and hot-swaps them locally.

Interactive Chat Slash Commands

When typing to ViClaw (either in the WebUI or CLI), you can bypass the LLM and issue direct system commands using the / prefix.

/clear

Instantly wipes the agent's short-term memory array. This resets the conversational context, essentially acting as an amnesia pill to stop hallucinations from looping.

/status

Returns the diagnostic state of the Agent, including active backend Model (e.g., Qwen 2.5), API connection latency, and the number of loaded Skills.

/skills

Prints an exhaustive list of every Python skill currently loaded into the Agent's tool execution framework, along with their JSON schema parameters.

/update

Triggers the OTA Updater Engine from within a chat natively, allowing you to patch the bot from a Telegram message.

Skills & ClawHub Marketplace

ViClaw operates heavily on Tools (Skills). By default, it cannot scrape websites or control Linux until it is given a Skill. Skills are python files stored in the /skills/ directory.

Dynamic Skill Resolver

If you ask ViClaw to do something it lacks the tool for, it will natively pause inference and ask you:
"To accomplish this, I need the `Web Scraper` skill from ClawHub. May I dynamically download and install it now?"
If you reply "yes", the agent pulls the python file directly from GitHub and injects it into its live execution cycle without restarting.

Auto-Dependency Healing

In Phase 18, ViClaw gained the ability to fix itself.

If the agent attempts to run a Python Skill, but it crashes because requests or bs4 isn't installed in the environment, the Agent loops the Python Stack Trace back into its reasoning engine. It will then pause and ask you for explicit permission to run pip install [package] or apt-get install [package].

Dashboards (3D & Tablet Kiosk)

The Web UI exposed on Port 8501 provides two distinct visual ways to monitor the agent.

The 3D Dashboard (/dashboard)

A monitoring-focused UI containing a ClawHub installation input form, System Log trailing, and an interactive Chat Box. It allows downloading forensic ZIP archives of the AI's internal actions.

The Stream Deck Kiosk (/kiosk)

A revolutionary, fullscreen, glassmorphic layout created specifically for iPads or Android tablets docked next to a PC. It features an animated WebGL 3D Robot that speaks agent statuses dynamically, backed by a changing CSS gradient bound to the actual atmospheric time of day. It contains interactive Pomodoro timers, clocks, and live iFrames.

Voice, Vision & Webhooks (Phase 21)

ViClaw now sees, hears, and speaks.

  • Continuous Ambient Voice: The 3D Kiosk utilizes the native Web Speech API to continuously transcribe your speech bridging local audio to Ollama, and utilizes text-to-speech to make the WebGL avatar talk.
  • Vision: Images attached to the CLI or the Dashboard's chat UI are locally Base64 string encoded and beamed to multimodal models (e.g. llava) bypassing data extraction layers.
  • Webhooks: Remote triggers and IoT devices (like Home Assistant) can POST raw JSON events to /api/webhook, violently forcing the AI into a "System Event" inference pass.

RAG Memory & Time Scheduling (Phase 22)

Memory Vault: Instead of forgetting you on reboot, ViClaw now extracts facts and auto-commits them to an internal SQLite Vector DB using the nomic-embed-text model. It performs mathematical numpy Cosine Similarity lookups prior to generating any response.

Natural Language Cron Engine

Tell the AI, "Every 15 minutes, scrape Google News and tell me the summary." A detached Python multithreading logic bomb (core/scheduler.py) tracks UTC timestamps and natively schedules Swarm generations independent of human triggering.

Swarm Architecture & Secure Auth (Phase 23)

ViClaw supports Login Context. The WebUI dashboard operates behind an encrypted HTTPOnly Cookie / JWT authentication gate mapping directly to your User Identifier.
