🚀 Installation

Requirements

Python 3.9+, git, and curl are all you need.

sudo apt update && sudo apt install -y python3 python3-venv git curl

One-Line Auto Deploy

bash <(curl -fsSL https://raw.githubusercontent.com/vignan07081999/ViClaw/main/setup.sh)

This command clones the repository, installs OS deps, builds the .venv sandbox, and boots the interactive Setup Wizard — which installs Ollama and pulls your selected models automatically.

Manual Clone

git clone https://github.com/vignan07081999/ViClaw.git
cd ViClaw && chmod +x install.sh && ./install.sh

One-Line Auto Un-Install

Wipe the daemon, python environment, and all databases entirely:

bash <(curl -fsSL https://raw.githubusercontent.com/vignan07081999/ViClaw/main/uninstall.sh)

🛠️ Skills & ClawHub Marketplace

ViClaw uses Skills (Tools) for all external actions. Skills are Python files in the skills/ directory that extend BaseSkill.

Skill What It Does
shell_engine Execute shell commands safely (MCP pattern)
file_io Read, write, and build files on disk
remote_ssh Run commands on remote servers via Paramiko
web_search DuckDuckGo scraper returning ranked results
homelab REST connector for HA, Sonarr, Radarr, Proxmox, etc.
reminders Persistent scheduled task memory
system_info CPU, RAM, disk, and uptime introspection
sessions Agent-to-agent delegation (Swarm)

Dynamic Skill Resolver

If ViClaw has no tool for a task, it pauses and queries ClawHub. You approve — it's injected live without a restart.

Hot Installs Phase 30

Skills installed via ClawHub at runtime now use delta loading — only the new file is imported. Existing skills keep their in-RAM runtime state untouched.

Auto-Dependency Healing

If a skill crashes from a missing Python package, ViClaw intercepts the stack trace, identifies the dependency, and asks your permission to pip install or apt-get install automatically.

🧠 Memory Architecture Phase 30

Layer Storage Behaviour
Short-term RAM + SQLite short_term_checkpoint Restored on daemon restart; isolated per session_id
Long-term SQLite memories table Cosine similarity via nomic-embed-text embeddings
RAG Vault SQLite + Ollama embeddings PDF/TXT chunks ingested via WebUI Document Vault
💾 Context Persistence: Your conversation survives daemon restarts. The short-term checkpoint is written to SQLite on every message and restored automatically on startup. Use /reset to clear it.

🔐 Security Model Phase 30

⚠️ Breaking Change: Hardcoded admin/claw credentials have been completely removed. You must run the install wizard to set credentials before the WebUI login will work.
What How
Credentials Set during conf_webui() wizard step — never hardcoded
Storage data/config.json → webui.credentials (gitignored)
Session cookies httponly=True, samesite=lax, 24h expiry
Session cleanup Expired sessions evicted hourly by background thread
Login audit All auth attempts logged with username and outcome
venv enforcement All entry points os.execv into .venv/bin/python3

To rotate credentials: run viclaw → [6] Update Configuration.

⚙️ Config Manager Phase 30

core/config.py now has a thread-safe ConfigManager class with live reload:

from core.config import reload_config
reload_config() # re-reads config.json from disk — no daemon restart needed

All accessor functions (is_webui_enabled(), get_webui_port(), etc.) and the APP_CONFIG alias are preserved for full backward compatibility.

💓 Proactive Heartbeat & Swarm Phase 30

The heartbeat loop wakes the LLM every idle ~5 minutes. It can:

  • Send spontaneous reminders if pending reminders exist
  • Execute proactive tool calls (check server health, etc.)
  • Broadcast to all enabled platform connectors — not just CLI (fixed in Phase 30)

The Swarm Orchestrator (core/swarm.py) deploys constrained child agents that run sub-tasks with isolated memory and permissions, then report back to the primary agent.

☁️ OTA Updates

ViClaw includes a background update engine (core/updater.py):

  • Compares local vs remote Git HEAD SHA
  • Stashes local changes before pulling
  • Updates pip dependencies if requirements.txt changed
  • Configured frequency: hourly / daily / weekly / manual

Trigger manually: viclaw → [10] OTA Updates.

rm -rf ~/ViClaw
